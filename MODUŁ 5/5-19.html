KONSTRUKTOR KLASY-CÓRKI Pierwsze, co należy zrobić w konstruktorze klasy-córki, to wywołać specjalną funkcję super(argumenty) - to pseudonim konstruktora klasy-rodzica. W przeciwnym razie, przy próbie zwrócenia się do this w konstruktorze klasy-córki wystąpi błąd. Przy wywołaniu konstruktora klasy-rodzica przekazujemy niezbędne mu argumenty dla inicjalizacji właściwości. class User { constructor(email) { this.email = email; } get email() { return this.email; } set email(newEmail) { this.email = newEmail; } } class ContentEditor extends User { constructor({ email, posts }) { // Wywołanie konstruktora klasy-rodzica User super(email); this.posts = posts; } } const editor = new ContentEditor({ email: "mango@mail.com", posts: [] }); console.log(editor); // { email: 'mango@mail.com', posts: [] } console.log(editor.email); // 'mango@mail.com'
