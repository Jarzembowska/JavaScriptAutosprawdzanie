METODY STATYCZNE W klasie można zadeklarować nie tylko metody przyszłego egzemplarza, ale też metody dostępne tylko dla klasy – metody statyczne, które mogą być zarówno publiczne, jak i prywatne. Składnia zadeklarowania jest analogiczna do statycznych właściwości z wyjątkiem tego, że wartością będzie metoda. class User { static #takenEmails = []; static isEmailTaken(email) { return User.#takenEmails.includes(email); } #email; constructor({ email }) { this.#email = email; User.#takenEmails.push(email); } } const mango = new User({ email: "mango@mail.com" }); console.log(User.isEmailTaken("poly@mail.com")); console.log(User.isEmailTaken("mango@mail.com")); Charakterystyka metod statycznych polega na tym, że w czasie ich wywołania słowo kluczowe this odnosi się do samej klasy. To znaczy, że metoda statyczna może uzyskać dostęp do statycznych właściwości klasy, ale nie do właściwości egzemplarza. Jest to logiczne, ponieważ metody statyczne wywołuje sama klasa, a nie jej egzemplarze.
